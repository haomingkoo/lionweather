# LionWeather

Singapore weather intelligence — NEA real-time data, ML-powered rainfall forecasting with SHAP analysis, and animated radar.

Live: [weather.kooexperience.com](https://weather.kooexperience.com)

---

## What it does

- **Real-time conditions** — pulls from Singapore's [data.gov.sg](https://api-open.data.gov.sg) every 10 minutes: temperature, rainfall, humidity, wind speed
- **4-day official forecast** — NEA's 4-day outlook shown first; extended days filled with Open-Meteo (or ML predictions once trained)
- **Rain category labels** — each forecast day is labelled Dry / Rain / Heavy Rain / Thundery, aligned to NEA's 4 categories
- **Smart notifications** — browser push alerts only when rain starts or clears (no spam)
- **NEA area name snapping** — when you pin a location, it resolves to the nearest official NEA neighbourhood (Ang Mo Kio, Tampines, etc.)
- **Radar** — animated rain area radar from weather.gov.sg
- **ML Analysis Dashboard** — EDA, ACF/PACF, FFT spectral analysis, SHAP feature importance, confusion matrix, training loss curves

---

## How to use the app

1. **Open** [weather.kooexperience.com](https://weather.kooexperience.com) — a prompt will appear asking for your location
2. **Allow location** to get your current neighbourhood's weather automatically (stays in your browser, never sent to a server)
3. **Add locations** — type coordinates in the form, or drop a pin on the Map View; names snap to official NEA neighbourhood names
4. **View forecast** — click any location card to see the 4-day NEA forecast with rain category labels (Dry / Rain / Heavy Rain / Thundery), hourly breakdown, radar, and weather details
5. **Radar** — the Hourly Forecast section has a Map button that opens the live NEA rain area radar
6. **ML Dashboard** — the ML tab shows time-series analysis, SHAP feature importance, and model accuracy once training has been run
7. **Notifications** — allow browser notifications to get alerts when rain starts or clears at your tracked locations
8. **Map View** — see all your pinned locations on an interactive map with live conditions

## Running locally

### Backend

```bash
cd backend
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env   # add your NEA API key

uvicorn app.main:app --reload --port 8000
```

API docs at `http://localhost:8000/docs`

### Frontend

```bash
cd frontend
npm install
npm run dev
```

App at `http://localhost:5173` — proxies `/api` to `localhost:8000`

## ML training workflow (local → Railway)

Training runs **locally** — the historical NEA dataset is too large for Railway:

```bash
cd backend
source venv/bin/activate

# Full analysis: EDA, ACF/PACF, FFT, SHAP, 4-class rain classifier
python -m ml.train_full_analysis

# Optional: NEA benchmark comparison
python -m ml.train_rainfall_forecast
```

This writes `backend/models/full_analysis.json`. Commit and push — Railway serves it via `/api/ml/full-analysis`.

**Rain categories (NEA-aligned):**
| Class | Label | Rainfall |
|-------|-------|---------|
| 0 | No Rain | < 0.1 mm/hr |
| 1 | Light Rain | < 7.6 mm/hr |
| 2 | Heavy Rain | < 30 mm/hr |
| 3 | Thundery Showers | ≥ 30 mm/hr |

**Training data:** NEA historical CSVs 2016–2024 in `backend/nea_historical_data/` (gitignored, ~46 GB).
Temporal split: train 2016–2022 / val 2023 / test 2024.

## Monitoring live data collection

Railway polls NEA every 10 minutes. Verify it's working:

```
GET /api/data-health/status    — health score, row counts, gap detection
GET /api/data-health/gaps      — specific time gaps (>15 min = warning)
GET /api/data-health/quality   — completeness, validity, duplicates
GET /api/data-health/timeline  — records per hour for last 7 days
GET /api/ml/data-sanity        — per-country row counts, missing fields
```

Full interactive API reference at `/docs`.

## Deployment

Both services deploy from the same repo on push to `main` via `railway.toml`:

- Frontend: `npm run build` → static files
- Backend: `uvicorn app.main:app` via `Procfile`

CORS is configured in `backend/app/main.py` to allow `https://weather.kooexperience.com`.
